# 数据库设计

## 约定

- **主键**：BIGSERIAL（自增 8 字节整数），用户 ID 的字符串形式直接作为工作区目录名称
- **时间类型**：`TIMESTAMP`（无时区），代码层统一使用 UTC
- **删除策略**：全表统一逻辑删除，新增 `deleted_at TIMESTAMP` 字段，标记删除时不释放磁盘。删除操作仅对文件系统生效（定制项目目录级联物理删除）。

---

## 实体关系总览

```
users 1──N custom_projects 1──N code_repos
  │                              │
  │                              └──（目录内实体，无 DB 关联）
  │
  ├── 1──N ssh_keys
  │
  └── 1──N conversations 1──N messages

custom_projects 1──N event_logs
```

## 表定义

### 1. users

| 字段 | 类型 | 约束 | 说明 |
|---|---|---|---|
| id | BIGSERIAL | PK | 主键，同时作为工作区目录名 |
| username | VARCHAR(100) | UNIQUE, NOT NULL | 登录账号 |
| alias_name | VARCHAR(100) | | 用户真实姓名，UI 优先展示，为空时回退 username |
| password_hash | VARCHAR(255) | NOT NULL | bcrypt 哈希值 |
| role | VARCHAR(20) | NOT NULL, CHECK(role IN ('admin','user')) | 角色 |
| created_at | TIMESTAMP | NOT NULL, DEFAULT NOW() | 创建时间 |
| deleted_at | TIMESTAMP | | 逻辑删除时间，NULL 表示未删除 |

### 2. custom_projects

| 字段 | 类型 | 约束 | 说明 |
|---|---|---|---|
| id | BIGSERIAL | PK | 主键 |
| user_id | BIGINT | NOT NULL, FK → users(id) | 所属用户 |
| name | VARCHAR(200) | NOT NULL | 项目名称 |
| description | TEXT | | 项目描述 |
| created_at | TIMESTAMP | NOT NULL, DEFAULT NOW() | 创建时间 |
| deleted_at | TIMESTAMP | | 逻辑删除时间 |

索引：`(user_id)` — 按用户查询项目列表。  
查询条件：`WHERE deleted_at IS NULL`。

### 3. code_repos

| 字段 | 类型 | 约束 | 说明 |
|---|---|---|---|
| id | BIGSERIAL | PK | 主键 |
| custom_project_id | BIGINT | NOT NULL, FK → custom_projects(id) | 所属定制项目 |
| name | VARCHAR(200) | NOT NULL | 仓库名称（从 URL 推断或用户指定） |
| url | VARCHAR(500) | NOT NULL | Git 远程仓库 URL |
| local_path | VARCHAR(500) | NOT NULL | 在定制项目目录下的相对路径 |
| current_branch | VARCHAR(200) | NOT NULL, DEFAULT 'main' | 当前所在本地分支 |
| created_at | TIMESTAMP | NOT NULL, DEFAULT NOW() | 创建时间 |
| deleted_at | TIMESTAMP | | 逻辑删除时间 |

索引：`(custom_project_id)`。

### 4. ssh_keys

| 字段 | 类型 | 约束 | 说明 |
|---|---|---|---|

| deleted_at | TIMESTAMP | | 逻辑删除时间 |

索引：`(user_id)` — 一个用户对应一套 SSH 密钥。

### 5. conversations

| 字段 | 类型 | 约束 | 说明 |
|---|---|---|---|
| id | BIGSERIAL | PK | 主键 |
| user_id | BIGINT | NOT NULL, FK → users(id) | 发起用户 |
| custom_project_id | BIGINT | NOT NULL, FK → custom_projects(id) | 关联定制项目 |
| title | VARCHAR(200) | | 对话标题（可由 AI 自动生成） |
| created_at | TIMESTAMP | NOT NULL, DEFAULT NOW() | 创建时间 |
| updated_at | TIMESTAMP | NOT NULL, DEFAULT NOW() | 最后消息时间 |
| deleted_at | TIMESTAMP | | 逻辑删除时间 |

### 6. messages

| 字段 | 类型 | 约束 | 说明 |
|---|---|---|---|
| id | BIGSERIAL | PK | 主键 |
| conversation_id | BIGINT | NOT NULL, FK → conversations(id) | 所属对话 |
| role | VARCHAR(20) | NOT NULL, CHECK(role IN ('user','assistant','tool','system')) | 消息角色 |

| tool_calls | JSONB | | assistant 消息：工具调用数组 |
| tool_call_id | VARCHAR(100) | | tool 消息：对应的工具调用 ID |

### 7. event_logs

| 字段 | 类型 | 约束 | 说明 |
|---|---|---|---|
| id | BIGSERIAL | PK | 主键 |

| supplement | TEXT | | 用户补充说明 |
| file_path | VARCHAR(500) | | 生成的 Markdown 文件路径 |
| created_at | TIMESTAMP | NOT NULL, DEFAULT NOW() | 创建时间 |
| deleted_at | TIMESTAMP | | 逻辑删除时间 |

索引：`(custom_project_id, created_at)` — 按项目倒序查看事件记录。

## 外键约束汇总

所有外键仅声明关联关系，**不含 ON DELETE 行为**（逻辑删除时代理由应用层处理）。

| 外键 | 来源 | 目标 |
|---|---|---|
| user_id | custom_projects | users |
| custom_project_id | code_repos | custom_projects |
| user_id | ssh_keys | users |
| user_id | conversations | users |
| custom_project_id | conversations | custom_projects |
| conversation_id | messages | conversations |
| custom_project_id | event_logs | custom_projects |
| user_id | event_logs | users |
| conversation_id | event_logs | conversations |

---

## 设计要点说明

1. **BIGSERIAL 自增主键**：低数据量低并发场景下够用，用户 ID 的字符串形式直接作为目录名，简洁清晰
2. **alias_name**：用于展示用户真实姓名，为空时 UI 回退到 username
3. **TIMESTAMP 无时区**：项目运行在统一时区（Docker Compose），代码层使用 `datetime.utcnow()` 写入 UTC 时间
4. **全表逻辑删除**：`deleted_at` 标记代替物理删除。所有业务查询附带 `WHERE deleted_at IS NULL`。恢复时将 `deleted_at` 置空即可
5. **外键无 ON DELETE 动作**：逻辑删除不会实际删除行，级联由应用层在标记删除时统一处理
6. **JSONB 存储 tool_calls**：DeepSeek API 的工具调用结构动态变化，JSONB 避免硬编码 schema
7. **ssh_keys 存指纹**：用于前端展示密钥标识，避免直接暴露文件路径
