## 1. 现状摘要

### 当前技术栈

| 层面 | 技术 | 备注 |
|------|------|------|
| 框架 | Vue 3.4 + TypeScript 5.3 | Composition API / `<script setup>` |
| 构建 | Vite 5 | — |
| 路由 | Vue Router 4 | HTML5 history mode |
| 状态 | Pinia | 仅存一个 auth store |
| UI 库 | Element Plus | 全量引入，未按需加载 |
| HTTP | Axios | 自定义拦截器 |
| 样式 | Scoped CSS | 无全局样式表，无 Tailwind |

### 当前页面清单

| 页面 | 路径 | 功能 |
|------|------|------|
| Login | `/login` | 用户登录 |
| Register | `/register` | 用户注册 |
| AppLayout | `/` | 三栏 IDE 布局外壳 |
| ProjectList | `/projects` | 左侧栏：项目列表 + 创建 |
| ProjectDetail | `/projects/:id` | **桩页面**（仅一行文字） |
| FilePanel | — | 中栏：文件树浏览 |
| DocEditor | — | 右栏：纯文本 textarea 编辑器 |
| ChatPanel | — | 中栏：会话列表 |
| ChatView | — | 右栏：SSE 流式 AI 对话 |
| RepoManager | — | 中栏：Git 仓库管理 |

### 存在的问题

- 全量引入 Element Plus（打包体积大）
- 无全局样式体系，各组件硬编码颜色值
- 无响应式设计，固定 220px + 260px 侧栏
- `marked` / `highlight.js` 已安装但未启用，消息以纯文本渲染
- `ProjectDetail.vue` 为桩页面
- SSH Key、Project 编辑/删除等 API 已定义但无 UI
- 用户信息未在页面刷新后恢复（token 存 localStorage 但未调 `/auth/me`）
- 无移动端适配

## 2. 框架选型

### 2.1 选型总览

| 层面 | 选型 | 版本 | 选型理由 |
|------|------|------|----------|
| **框架** | React | 19.2.x | 最新稳定版，`useEffectEvent` 解决 SSE 闭包陈旧问题，`<Activity>` 预渲染隐藏路由 |
| **语言** | TypeScript | 5.9.x | 稳定可靠，严格模式，路径别名 `@/` |
| **构建** | Vite | 7.x | 已有使用经验，React 官方推荐，稳定 |
| **路由** | React Router | 7.x（声明式模式） | 与 Vue Router 心智模型一致，SPA 声明式路由足够 |
| **服务端状态** | TanStack Query | v5 | 缓存、后台刷新、乐观更新、重试、去重 |
| **客户端状态** | Zustand | v5 | ~1KB，无 Provider 包裹，`persist` 中间件同步 localStorage |
| **UI 组件** | shadcn/ui | Latest | 基于 Radix UI + Tailwind CSS，代码复制到项目中，完全可控 |
| **样式** | Tailwind CSS | v4 | CSS-first 配置，LightningCSS 加速，零配置自动检测 |
| **图标** | Lucide React | Latest | shadcn/ui 默认图标库，风格统一 |
| **表单** | React Hook Form + Zod | v7 + Latest | 非受控模式高性能，Zod schema 校验 |
| **HTTP** | 自封装 fetch wrapper | — | ~50 行，支持 JWT 注入、401 拦截、错误规范化 |
| **Markdown 渲染** | react-markdown + rehype-highlight | Latest | 流式消息的 Markdown 渲染 + 代码高亮 |
| **Toast 通知** | Sonner | Latest | shadcn/ui 默认方案，轻量美观 |
| **工具类** | clsx + tailwind-merge | Latest | 条件类名 + Tailwind 类冲突合并 |

### 2.2 不选的方案及原因

| 方案 | 不选原因 |
|------|----------|
| Next.js / Remix | 项目有独立 FastAPI 后端，不需要 SSR / SSG，纯 SPA 即可 |
| Ant Design 5 | 与 Tailwind CSS 风格冲突，定制成本高，打包体积大 |
| MUI (Material UI) | Material Design 风格与产品定位不符，包体积大 |
| Redux Toolkit | 对于当前规模过重，Zustand 更轻量 |
| TanStack Router | 学习曲线陡峭，主要优势（类型安全 loader）可通过 TanStack Query 获得 |
| Axios | 2025 年 fetch 生态已成熟，原生 fetch + thin wrapper 更轻量 |

### 目录设计原则

- **`components/ui/`**：shadcn/ui 自动生成，**禁止手动修改命名或删除**，可通过 `npx shadcn add` 追加
- **`components/shared/`**：跨页面复用的业务组件
- **`pages/`**：按路由组织，每个页面一个文件，复杂页面内含 `components/` 子目录
- **`api/`** vs **`queries/`**：分离关注点 — `api/` 负责 HTTP 通信，`queries/` 负责缓存/轮询/乐观更新
- **`stores/`**：仅存放客户端状态（非服务端数据），服务端数据由 TanStack Query 管理

## 4. 设计原则

### 4.1 总体理念

> **工具型产品，效率优先。** Tech Assistant 是面向开发者的 AI 编程助手，界面应传递「专业、高效、可信赖」的气质，而非花哨或娱乐化。

### 4.2 五项核心原则

#### 原则一：信息层级清晰

- 任何页面，用户在 **3 秒内** 能识别当前上下文（哪个项目？哪个对话？）
- 一个页面只有一个**主要操作**（Primary Action），次要操作视觉降级
- 使用大小、粗细、颜色三重手段建立层级，不依赖单一维度

#### 原则二：一致性优先于创意

- 同类元素（按钮、输入框、卡片）在所有页面保持相同样式
- 间距、圆角、阴影使用统一 token，不随意写魔数
- 交互模式（hover、focus、loading）全局统一

#### 原则三：留白即信息

- 不用线条分隔内容，用**间距**分隔
- 组件之间的距离 > 组件内部的距离 > 元素之间的距离
- 页面边缘留白 ≥ 24px

#### 原则四：克制用色

- 主色仅用于：**关键操作按钮、链接、选中态、品牌标识**
- 中性色（gray scale）承担 80% 以上的视觉面积
- 语义色（success / warning / error）仅用于状态反馈

#### 原则五：响应即信任

- 每个用户操作必须在 **100ms 内** 给出视觉反馈
- 耗时操作显示 loading 态并禁用重复触发
- 错误信息具体、可操作，不使用「发生了错误」

## 5. 设计语言

### 5.1 色彩系统

基于 Tailwind CSS v4 的 `@theme` 定义，所有颜色使用 OKLCH 色彩空间以确保感知均匀。

#### 5.1.1 中性色（Neutral）

用于文本、背景、边框、分割线，构成界面的 80% 视觉面积。

| Token | 用途 | 亮色模式值 |
|-------|------|-----------|
| `--color-background` | 页面底色 | `oklch(0.99 0 0)` — 近白 |
| `--color-foreground` | 正文文本 | `oklch(0.15 0 0)` — 深灰（非纯黑） |
| `--color-muted` | 次要文本、占位符 | `oklch(0.55 0 0)` |
| `--color-muted-foreground` | 次要文本色 | `oklch(0.50 0 0)` |
| `--color-card` | 卡片背景 | `oklch(1 0 0)` — 纯白 |
| `--color-border` | 边框、分割线 | `oklch(0.92 0 0)` |
| `--color-input` | 输入框边框 | `oklch(0.88 0 0)` |
| `--color-accent` | 悬停背景、轻微强调 | `oklch(0.96 0 0)` |

> **禁止使用纯黑 `#000000`**。正文使用 `oklch(0.15 0 0)`（约 `#1A1A1A`），保证与白色背景对比度 ≥ 15:1。

#### 5.1.2 主色（Primary）

用于品牌标识、关键操作按钮、选中状态、链接。

| Token | 值 | 说明 |
|-------|-----|------|
| `--color-primary` | `oklch(0.55 0.18 250)` | 蓝色调，专业感 |
| `--color-primary-foreground` | `oklch(0.98 0 0)` | 主色上的文本（白色） |
| `--color-primary-hover` | `oklch(0.50 0.18 250)` | 主色悬停态（加深 5%） |

> 主色在页面中占比 ≤ 5%。仅用于：提交按钮、发送按钮、选中行高亮、链接文字、品牌 logo。

#### 5.1.3 语义色（Semantic）

用于状态反馈，不用于装饰。

| Token | 值 | 用途 |
|-------|-----|------|
| `--color-success` | `oklch(0.62 0.17 155)` | 成功状态、完成标记 |
| `--color-warning` | `oklch(0.75 0.15 75)` | 警告、进行中、思考中 |
| `--color-destructive` | `oklch(0.55 0.2 25)` | 错误、删除、危险操作 |
| `--color-info` | `oklch(0.65 0.15 230)` | 信息提示 |

#### 5.1.4 对比度要求

- 正文文本与背景：≥ **7:1**（WCAG AAA）
- 大标题与背景：≥ **4.5:1**（WCAG AA）
- 辅助文本与背景：≥ **4.5:1**
- 按钮文本与按钮背景：≥ **4.5:1**

#### 5.1.5 暗色模式预留

色彩系统基于 OKLCH + CSS 变量，暗色模式通过 `dark` class 切换即可实现。初期不实现，但架构上保证可扩展性：

```css
.dark {
  --color-background: oklch(0.13 0 0);
  --color-foreground: oklch(0.93 0 0);
  --color-card: oklch(0.17 0 0);
  /* ... */
}
```

### 5.2 排版系统

#### 5.2.1 字体栈

```css
--font-sans: "Inter", -apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif;
--font-mono: "JetBrains Mono", "Fira Code", ui-monospace, "SF Mono", monospace;
```

- **正文 / UI 文本**：Inter（Google Fonts CDN 或 self-hosted）
- **代码 / 终端**：JetBrains Mono（等宽字体，用于代码编辑器、工具调用输出）

#### 5.2.2 字号阶梯

基于 **8px 基准网格**，采用 1.25 倍递增比（Major Third）：

| Token | 字号 | 行高 | 用途 |
|-------|------|------|------|
| `--text-xs` | 12px | 16px | 辅助标签、时间戳 |
| `--text-sm` | 14px | 20px | 次要文本、表格内容 |
| `--text-base` | 16px | 24px | 正文 |
| `--text-lg` | 18px | 28px | 小标题、卡片标题 |
| `--text-xl` | 20px | 28px | 页面副标题 |
| `--text-2xl` | 24px | 32px | 页面标题 |
| `--text-3xl` | 30px | 36px | 首屏大标题（极少使用） |

> **标题字号至少比正文大 50%**。正文 16px → 标题最小 24px。

#### 5.2.3 字重

| 字重 | 值 | 用途 |
|------|-----|------|
| Regular | 400 | 正文 |
| Medium | 500 | 按钮文本、标签、导航项 |
| Semibold | 600 | 小标题、卡片标题 |
| Bold | 700 | 页面标题 |

### 5.3 间距系统

所有间距为 **8px 的倍数**，不允许使用 8 的非整数倍值（如 6px、10px、14px）。

| Token | 值 | 典型用途 |
|-------|-----|----------|
| `--spacing-1` | 4px | 图标与文本间距（唯一例外：4px = 8/2） |
| `--spacing-2` | 8px | 元素内部紧凑间距 |
| `--spacing-3` | 12px | 输入框内边距 |
| `--spacing-4` | 16px | 组件内边距、列表项间距 |
| `--spacing-5` | 20px | 卡片内边距 |
| `--spacing-6` | 24px | 页面边缘留白、区块间距 |
| `--spacing-8` | 32px | 大区块间距 |
| `--spacing-10` | 40px | 页面顶部留白 |
| `--spacing-12` | 48px | 区域之间分隔 |
| `--spacing-16` | 64px | 页面底部留白 |

**层级关系**：

```
元素间距 (8px) < 组件内边距 (16px) < 组件间距 (24px) < 区块间距 (32-48px)
```

### 5.4 圆角系统

| Token | 值 | 用途 |
|-------|-----|------|
| `--radius-sm` | 4px | 小元素（Badge、小按钮） |
| `--radius-md` | 8px | 输入框、普通按钮、卡片 |
| `--radius-lg` | 12px | 大卡片、对话气泡、模态框 |
| `--radius-full` | 9999px | 头像、圆形按钮 |

### 5.5 阴影系统

基于 Tailwind 内置阴影，定义三级阴影：

| Token | 值 | 用途 |
|-------|-----|------|
| `--shadow-sm` | `0 1px 2px oklch(0 0 0 / 0.05)` | 按钮、输入框浮起感 |
| `--shadow-md` | `0 4px 6px -1px oklch(0 0 0 / 0.08)` | 卡片、下拉菜单 |
| `--shadow-lg` | `0 10px 15px -3px oklch(0 0 0 / 0.1)` | 模态框、浮动面板 |

> 阴影用于层级暗示，不用于装饰。同一视口内最多出现 3 个阴影层级。

### 5.6 动效规范

| 属性 | 时长 | 缓动 | 用途 |
|------|------|------|------|
| 颜色变化 | 150ms | `ease-in-out` | hover/focus 颜色过渡 |
| 尺寸变化 | 200ms | `ease-out` | 展开/折叠、高度变化 |
| 位移 | 200ms | `ease-out` | 滑入/滑出 |
| 弹窗 | 200ms | `ease-out` | Dialog / Sheet 进入 |
| 骨架屏 | 1.5s | `ease-in-out` 循环 | shimmer 动画 |

> 所有动画必须可通过 `prefers-reduced-motion` 媒体查询禁用。

```typescript

  projects: ['projects'] as const,
  project: (id: number) => ['projects', id] as const,
  files: (projectId: number) => ['projects', projectId, 'files'] as const,
  file: (projectId: number, path: string) => ['projects', projectId, 'files', path] as const,
  conversations: (projectId: number) => ['projects', projectId, 'conversations'] as const,
  conversation: (id: number) => ['conversations', id] as const,
  repos: (projectId: number) => ['projects', projectId, 'repos'] as const,
```

### 8.4 应用启动流程

```
1. main.tsx 渲染 App.tsx
2. App.tsx 读取 auth-store（从 localStorage 恢复 token）
3. 若 token 存在 → 调 GET /auth/me 获取用户信息 → 写入 auth-store
   若 token 不存在 → 跳转 /login
4. /auth/me 返回 401 → 清除 token → 跳转 /login
5. 用户信息就绪 → 渲染路由
```

> 这解决了当前 Vue 版本「刷新页面后用户信息丢失」的问题。

### 9.1 Fetch Wrapper (`lib/api-client.ts`)

```typescript
// 核心能力：
// 1. 自动附加 Authorization: Bearer <token>（从 auth-store 读取）
// 2. 统一 JSON 解析 + 错误规范化
// 3. 401 响应自动清除 token + 重定向 /login

interface ApiResponse<T> {

  message: string
  data: T
}

// data 类型为 Project[]
```

### 9.2 API 函数设计

```typescript
// api/projects.ts — 纯函数，不做缓存
export const projectApi = {
  list: () => client<Project[]>('/api/projects'),
  get: (id: number) => client<Project>(`/api/projects/${id}`),
  create: (data: CreateProjectDto) => client<Project>('/api/projects', { method: 'POST', body: data }),
  update: (id: number, data: UpdateProjectDto) => client<Project>(`/api/projects/${id}`, { method: 'PUT', body: data }),
  delete: (id: number) => client<void>(`/api/projects/${id}`, { method: 'DELETE' }),
}

// queries/use-projects.ts — TanStack Query hooks
export function useProjects() {
  return useQuery({ queryKey: queryKeys.projects, queryFn: projectApi.list })
}

export function useCreateProject() {
  const queryClient = useQueryClient()
  return useMutation({
    mutationFn: projectApi.create,
    onSuccess: () => queryClient.invalidateQueries({ queryKey: queryKeys.projects }),
  })
}
```

### 12.2 响应式布局方案

| 断点 | 布局 |
|------|------|
