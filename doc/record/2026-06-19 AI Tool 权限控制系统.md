## 1. 背景与需求

### 现状

AI 对话中，DeepSeek 模型返回 `tool_calls` 后，`AiClient._execute_tool()` **无条件直接执行**，没有任何权限检查。6 个现有 Tool：

| Tool | 风险 |
|:---|:---|
| `run_command` | 高 |
| `write_file` | 中 |
| `delete_file` | 高 |
| `read_file` | 低 |
| `search_content` | 低 |
| `list_directory` | 低 |

### 需求

为每个 Tool 配置三态策略：
- **auto_approve**：直接执行
- **ask_user**：向用户展示审批弹窗，确认后执行
- **deny**：直接拒绝，返回错误给 AI

## 2. 方案设计

### 权限层级

```
全局默认（硬编码）
  ↓ 覆盖
项目配置（tool_permission_configs 表）
  ↓ 覆盖
对话内存覆盖（"本对话始终允许"，不持久化）
```

### 中断机制

当 Tool 处于 `ask_user` 状态时，使用 `asyncio.Event` 暂停 Agentic Loop：

```
AiClient.stream()
  ├─ AI 返回 tool_calls
  ├─ 查权限 → ask_user
  ├─ 创建 ApprovalSlot + asyncio.Event
  ├─ yield SSE: tool_approval_required
  ├─ await event.wait(timeout=120)
  │     │
  │     │ 前端弹出审批弹窗
  │     │ 用户操作 → POST /api/conversations/{id}/tool-approval
  │     │ 后端 approval.approve() → event.set()
  │     │
  ├─ 检查 decision → 执行/拒绝
  └─ 继续下一轮 Agentic Loop
```

### 全局默认

| Tool | 默认策略 |
|:---|:---|
| `read_file` | `auto_approve` |
| `search_content` | `auto_approve` |
| `list_directory` | `auto_approve` |
| `write_file` | `ask_user` |
| `run_command` | `ask_user` |
| `delete_file` | `ask_user` |

## 3. 数据库变更

### 新表：`tool_permission_configs`

```sql
CREATE TABLE tool_permission_configs (
    id              BIGSERIAL       PRIMARY KEY,
    project_id      BIGINT          NOT NULL REFERENCES custom_projects(id),
    tool_name       VARCHAR(50)     NOT NULL CHECK (tool_name IN (
                        'run_command', 'read_file', 'write_file',
                        'search_content', 'list_directory', 'delete_file'
                    )),
    permission      VARCHAR(20)     NOT NULL CHECK (permission IN (
                        'auto_approve', 'ask_user', 'deny'
                    )),
    created_at      TIMESTAMP       NOT NULL DEFAULT NOW(),
    updated_at      TIMESTAMP       NOT NULL DEFAULT NOW(),
    deleted_at      TIMESTAMP
);

CREATE UNIQUE INDEX idx_tool_permission_configs_project_tool
    ON tool_permission_configs(project_id, tool_name);
```

## 5. SSE 协议扩展

### 新增事件类型

**审批请求**（AiClient → 前端）：
```
event: tool_approval_required
data: {
  "tool_call_id": "call_abc123",
  "tool_name": "write_file",
  "arguments": {"file_path": "src/app.py", "content": "..."}
}
```

**工具被拒绝**（AiClient → 前端）：
```
event: tool_denied
data: {
  "tool_call_id": "call_abc123",
  "tool_name": "write_file",
  "reason": "user_denied" | "policy_deny"
}
```

## 7. 审批流程时序

```
用户发送消息
  │
  ▼
POST /conversations/{id}/stream
  │
  ▼
AiClient.stream()
  │
  ▼
DeepSeek 返回 tool_calls: [write_file, read_file]
  │
  ├─ read_file → auto_approve → 直接执行 → yield tool_result
  │
  ├─ write_file → ask_user
  │   ├─ 创建 ApprovalSlot
  │   ├─ register(conv_id, slot)
  │   ├─ yield SSE: tool_approval_required
  │   │     │
  │   │     ▼
  │   │  前端 ToolApprovalDialog 弹出
  │   │  用户点击"允许本次"
  │   │     │
  │   │     ▼
  │   │  POST /conversations/{id}/tool-approval
  │   │  {tool_call_id, decision: "approved", scope: "once"}
  │   │     │
  │   │     ▼
  │   │  approval.approve() → event.set()
  │   │
  │   ├─ await event.wait() → ("approved", "once")
  │   ├─ _run_tool() → 执行
  │   └─ yield tool_result
  │
  ▼
继续下一轮 Agentic Loop...
  │
  ▼
finally: approval_mgr.cleanup(conv_id)
```
