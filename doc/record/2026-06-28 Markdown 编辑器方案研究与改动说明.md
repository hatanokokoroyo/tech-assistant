# Markdown 编辑器方案研究与改动说明

## 背景

当前项目的文件编辑器（`FileEditorContent`）对所有文件类型统一使用原生 `<textarea>` 进行编辑，包括 `.md` 文件。这意味着 Markdown 文件没有语法高亮、没有实时预览、可读性和编辑便利性不足。

### 现有实现

- **文件编辑器组件**：`frontend/src/pages/app/file-panel.tsx` 中的 `FileEditorContent`
- **编辑器 Hook**：`frontend/src/pages/app/file-editor-hook.ts` 中的 `useFileEditor`
- **当前技术**：原生 `<textarea>`，无语法高亮，无 `.md` 特殊处理
- **已有依赖**：`react-markdown` v10.1.0 + `remark-gfm` v4.0.1（仅用于聊天消息渲染，未用于文件编辑器）

## 方案研究

### 对比表

| 方案 | npm 周下载 | Bundle (gzip) | 模式 | 语法高亮 | 工具栏 | 预览 | 维护状态 | React 19 |
|------|-----------|---------------|------|---------|--------|------|---------|---------|
| **CodeMirror 6 + react-markdown** | 3.7M + 25.4M | ~83KB | 分栏编辑 | ✅ 优秀 | ❌ 需自建 | ✅ 分栏预览 | 活跃 | ✅ |
| **@uiw/react-md-editor** | 686K | ~361KB | 分栏编辑 | ✅ 基础 | ✅ 内置 | ✅ 内置 | 活跃 | ⚠️ 待验证 |
| **Milkdown** | 221K | ~93KB | WYSIWYG | ⚠️ 有限 | ✅ 插件 | WYSIWYG 即预览 | 活跃 | ✅ |
| **Tiptap** | 10.2M | ~127KB+ | WYSIWYG | ⚠️ 有限 | ✅ 内置 | WYSIWYG 即预览 | 活跃 | ✅ |
| **ByteMD** | 7.5K | ~132KB | 分栏编辑 | ✅ | ✅ 内置 | ✅ | ⚠️ 停更 | ❌ |

### 选定方案：CodeMirror 6 + react-markdown 分栏

**选择理由：**

1. **IDE 定位匹配** — 产品是开发者工作台，用户期望 VS Code 级别的 Markdown 编辑体验
2. **最小侵入** — 项目已有 `react-markdown`，只需新增 `@uiw/react-codemirror` + `@codemirror/lang-markdown`
3. **Bundle 最轻** — ~83KB gzip，比 @uiw/react-md-editor（361KB）小 4 倍
4. **样式完全可控** — headless 组件，完美兼容 Tailwind + shadcn/ui 设计系统
5. **可渐进实现** — 先用 CM6 替换 textarea，再加分栏预览

**淘汰的方案：**

- **@uiw/react-md-editor**：Bundle 太大（361KB），样式定制困难，与 Tailwind/shadcn 有冲突
- **Tiptap**：Markdown 是二等公民，WYSIWYG 模式隐藏源码，不适合 `.md` 文件编辑
- **Milkdown**：WYSIWYG 模式隐藏源码，开发者用户可能不习惯
- **ByteMD**：已事实停更，周下载仅 7.5K

## 改动说明

### 预期改动文件

| 文件 | 改动类型 | 说明 |
|------|---------|------|
| `frontend/src/pages/app/file-panel.tsx` | 修改 | `FileEditorContent` 中根据文件扩展名判断是否为 `.md`，若是则渲染分栏编辑器 |
| `frontend/src/components/ui/markdown-editor.tsx` | 新建 | 封装 CodeMirror 编辑器 + react-markdown 预览的分栏组件 |
| `frontend/package.json` | 修改 | 新增依赖 `@uiw/react-codemirror`、`@codemirror/lang-markdown` |

### 改动效果

- `.md` 文件获得 Markdown 语法高亮、行号、代码折叠等 IDE 级编辑体验
- 右侧实时渲染预览，支持表格、任务列表、代码块高亮等 GFM 特性
- 其他文件类型（`.py`、`.json` 等）保持原有 textarea 编辑方式，不受影响

### 实施路径

```
Phase 1: CM6 替换 textarea（最低成本，立即改善）
  → 修改 FileEditorContent，.md 文件用 CodeMirror + lang-markdown
  → 其他文件保持 textarea 或后续统一迁移

Phase 2: 加入分栏预览（可选增强）
  → 右侧加 react-markdown 预览面板
  → 支持滚动同步
  → 可切换「仅编辑 / 分栏 / 仅预览」模式
```
