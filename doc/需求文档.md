# 需求文档 — Tech Assistant

## 1. 项目概述

### 1.1 背景

公司为多家甲方开发定制项目，每个定制项目下包含多个代码项目，均已部署至客户生产服务器。客户时常提出技术支持需求（如线上功能异常），研发需查阅代码并查询数据库以定位原因。由于原开发人员已离职且交接不充分，需要开发一个结合 AI 大模型的 Web 软件辅助研发解决问题。

### 1.2 目标

开发一套 Docker Compose 一键部署的 Web 应用，研发人员登录后可为每个甲方的定制项目创建 AI 会话，AI 可访问该项目下的代码仓库和相关文档，辅助进行问题定位和分析。

---

## 2. 技术架构

### 2.1 整体架构

```
用户浏览器
    │
    ├── HTTP (REST API) ──→ FastAPI Backend ──→ PostgreSQL
    │                                               ↑
    └── SSE (流式输出) ──→ FastAPI Backend ────────┘
                              │
                              ├── 调用 DeepSeek API
                              ├── 执行 Tool Calls（文件操作 / 命令）
                              └── 共享 Volume: /data/tech-assistant
```

AI 模块作为 backend 进程的一部分运行，不独立部署服务。

### 2.2 Docker Compose 服务

| 服务 | 镜像 | 说明 |
|---|---|---|
| `frontend` | Nginx + Vue 3 | 提供前端静态资源，反向代理后端 API |
| `backend` | Python + FastAPI | REST API + SSE 流式输出 + AI 模块（DeepSeek + Tool Calls） |
| `postgres` | PostgreSQL | 主数据库 |

所有服务均可访问外网。

### 2.3 技术栈

| 层 | 选型 | 说明 |
|---|---|---|
| 后端框架 | FastAPI | Python 异步 Web 框架 |
| 前端框架 | Vue 3 | Composition API |
| UI 组件库 | Element Plus | Vue 3 生态 |
| 数据库 | PostgreSQL | 容器内运行 |
| ORM | SQLAlchemy | FastAPI 生态配合 |
| 认证 | JWT + bcrypt | Token 鉴权 |
| AI SDK | openai | Python 库，对接 DeepSeek API |
| 包管理 | pip + requirements.txt（后端）/ npm（前端） |
| 部署 | Docker Compose | 一键构建和启动 |
| 配置 | `.env` 文件 | 环境变量注入容器 |

## 3. 功能模块

### 3.1 登录模块

- 支持管理员后台创建用户和用户自助注册（无需审核，部署在公司内网）
- 注册和登录仅需账号和密码
- 密码使用 **bcrypt** 加密存储
- 认证使用 **JWT Token**，请求时携带 Token

#### 角色体系

| 角色 | 权限 |
|---|---|
| 管理员 | 管理所有用户和所有定制项目 |
| 普通用户 | 管理自己的定制项目、文档和 AI 会话 |

### 3.2 文件管理系统

#### 3.2.1 相对根目录

Docker Compose 中声明 named volume：

```yaml
volumes:
  tech_assistant_data:
```

容器内挂载路径为 `/data/tech-assistant`，主机路径通过 `.env` 文件中的 `HOST_DATA_DIR` 配置。

#### 3.2.2 目录结构

```
/data/tech-assistant/
  ├── <user_id>/                     # 用户工作区，账号创建时自动生成
  │   ├── .ssh/                      # 用户 SSH 私钥目录
  │   │   └── id_rsa                 # 用户在前端配置页面上传
  │   ├── <custom_project>/          # 用户创建的定制项目
  │   │   ├── instructions.md         # 项目信息及文档索引（自动创建）
  │   │   ├── doc/
  │   │   │   ├── log/               # 事件处理记录（用户确认后生成）
  │   │   │   └── reference-doc/     # AI 自动保存的可参考文档
  │   │   ├── <code_repo_1>/         # git clone 的代码项目
  │   │   ├── <code_repo_2>/
  │   │   └── ...
  │   └── <custom_project_2>/
  └── ...
```

#### 3.2.3 目录生命周期

| 操作 | 行为 |
|---|---|
| 创建用户 | 自动创建 `<user_id>/` 工作区 |
| 创建定制项目 | 自动创建 `instructions.md` + `doc/` + `doc/log/` + `doc/reference-doc/` |
| 删除定制项目 | 级联删除整个项目目录 |
| 添加代码项目 | 用户提供 Git URL，系统 `git clone` 到定制项目目录下 |
| 删除代码项目 | 删除本地目录，不支持解绑保留 |

#### 3.2.4 Git 操作

- 用户在前端-用户设置中上传 SSH 秘钥，保存至用户工作区下的 `.ssh/`目录下
- 当用户在仓库中使用git clone命令时, 需要使用该秘钥进行克隆
- 通过 `GIT_SSH_COMMAND` 指定密钥执行 `git clone`
- 公钥由用户在 Git 平台（GitHub/GitLab）自行配置
- 用户均可查询远程分支和切换本地分支

### 3.3 AI 交互模块

#### 3.3.1 模型

- 语言：Python
- 库：openai
- 模型：DeepSeek V4 Flash / DeepSeek V4 Pro
- 协议：Tool Calls API

#### 3.3.2 Tool Calls 能力

AI 具备以下工具调用能力：

| 工具 | 能力范围 |
|---|---|
| 命令行 | 在 `/data/tech-assistant` 沙箱内执行命令 |
| 文件操作 | 浏览、创建、修改、删除 `/data/tech-assistant` 下的文件和目录 |
| Git 操作 | `git clone`、`git fetch`、`git branch`、`git checkout` 等 |

所有操作限制在 `/data/tech-assistant` 路径下。

#### 3.3.3 会话生命周期

会话绑定到用户所属的定制项目：

```
用户选择一个定制项目
    → 创建新对话（或加载历史对话）
    → 发起提问
    → Backend 调用 DeepSeek API
    → DeepSeek 返回流式结果（思维链 / 回复 / 工具调用）
    → Backend 通过 SSE 实时推送到前端
    → 对话中所有内容保存至 PostgreSQL
    → 用户可以随时重新打开历史对话继续
```

#### 3.3.4 流式输出规范

前端展示类似当前主流大模型对话服务：

- **回复内容**：直接展示
- **思维链**：默认折叠，可展开
- **工具调用**：默认折叠缩略，展示工具名称和调用状态（调用中 / 完成 / 失败）
- 所有内容随 SSE 事件流实时渲染

### 3.4 用户交互模块

#### 3.4.1 页面布局

```
┌─────────────────────────────────────────────────┐
│  ┌───────────┐  ┌─────────┐  ┌───────────────┐ │
│  │           │  │         │  │               │ │
│  │ 定制项目  │  │ 文件列表│  │  内容区       │ │
│  │ 列表      │  │ 或      │  │  (文件编辑器  │ │
│  │           │  │ 对话列表│  │   或          │ │
│  │           │  │         │  │   AI 对话界面)│ │
│  │           │  │         │  │               │ │
│  └───────────┘  └─────────┘  └───────────────┘ │
└─────────────────────────────────────────────────┘
```

- **第一层导航**（最左侧）：定制项目列表
- **第二层导航**（中间）：选中定制项目后的子导航（文件列表 / 对话列表），上方有返回按钮
- **内容区**（右侧）：文件编辑器（Markdown）或 AI 对话界面

#### 3.4.2 文件管理

- 用户可查看和编辑 `instructions.md`、`log/` 和 `reference-doc/` 下的文档
- 编辑器基于 Markdown 编辑器（Vditor 或 Marked），支持实时预览和编辑
- 代码项目不提供内部文件浏览能力

#### 3.4.3 分支管理

每个代码项目在列表中展示当前所在分支，右侧提供切换分支按钮：

- 点击按钮弹出分支列表（本地分支 + 远程分支）
- 选择分支后调用后端 API 执行切换

#### 3.4.4 事件处理完成确认

AI 处理完事件后，用户点击"完成"按钮，弹窗中可输入补充事项，确认后系统自动生成 Markdown 记录保存至 `doc/log/`。

## 4. 数据模型（概要）

### 主要表

| 表 | 字段概要 | 说明 |
|---|---|---|
| `users` | id, username, password_hash, role, created_at | 用户信息 |
| `custom_projects` | id, user_id, name, description, created_at | 定制项目 |
| `code_repos` | id, custom_project_id, name, url, local_path, current_branch | 代码项目 |
| `ssh_keys` | id, user_id, private_key_path, created_at | 用户 SSH 私钥记录 |
| `conversations` | id, user_id, custom_project_id, title, created_at, updated_at | AI 对话会话 |
| `messages` | id, conversation_id, role, content, tool_calls, created_at | 对话消息 |
| `event_logs` | id, custom_project_id, user_id, summary, supplement, file_path, created_at | 事件处理记录 |

## 5. Docker Compose 配置概要

```yaml
# docker-compose.yml（结构示意）
version: "3.9"
services:
  frontend:
    build: ./frontend
    ports: ["80:80"]
    depends_on: [backend]

  backend:
    build: ./backend
    env_file: .env
    volumes:
      - tech_assistant_data:/data/tech-assistant
    depends_on: [postgres]

  postgres:
    image: postgres:16
    env_file: .env
    volumes:
      - pg_data:/var/lib/postgresql/data

volumes:
  tech_assistant_data:
  pg_data:
```

### 环境变量（.env）

```
# 数据库
POSTGRES_DB=tech_assistant
POSTGRES_USER=postgres
POSTGRES_PASSWORD=your_password

# DeepSeek
DEEPSEEK_API_KEY=your_api_key

# JWT
JWT_SECRET_KEY=your_secret_key

# 数据目录（主机路径）
HOST_DATA_DIR=./data
```

## 6. 非功能性需求

| 需求 | 说明 |
|---|---|
| 沙箱安全 | AI 执行的命令和文件操作限制在 `/data/tech-assistant` 路径下 |
| 数据持久化 | 数据库和文件系统数据均通过 Docker volume 持久化 |
| 单点部署 | Docker Compose 一键启动所有服务 |
| 会话持久化 | AI 对话全量保存到 PostgreSQL，支持历史加载和继续 |
