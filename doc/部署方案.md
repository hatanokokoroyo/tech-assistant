# 部署方案

## Docker Compose 架构

```yaml
services:
  frontend:     # Nginx + Vue 静态资源，反向代理 /api → backend
  backend:      # FastAPI + AI 模块，对外 SSE 流式接口
  postgres:     # PostgreSQL 16
```

## Dockerfile：前端

```dockerfile
# frontend/Dockerfile
# Stage 1: build
FROM node:20-alpine AS builder
WORKDIR /app
COPY package.json ./
RUN npm install
COPY . .
RUN npm run build

# Stage 2: serve
FROM nginx:stable-alpine
COPY --from=builder /app/dist /usr/share/nginx/html
COPY nginx.conf /etc/nginx/conf.d/default.conf
EXPOSE 80
CMD ["nginx", "-g", "daemon off;"]
```

```nginx
# frontend/nginx.conf
server {
    listen       80;
    server_name  localhost;

    root   /usr/share/nginx/html;
    index  index.html;

    # 后端 API 反向代理
    location /api/ {
        proxy_pass http://backend:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;

        # SSE 长连接支持
        proxy_buffering off;
        proxy_cache off;
        proxy_read_timeout 300s;
    }

    # 前端路由（SPA fallback）
    location / {
        try_files $uri $uri/ /index.html;
    }
}
```

## Dockerfile：后端

```dockerfile
# backend/Dockerfile
FROM python:3.12-slim

WORKDIR /app

RUN apt-get update && apt-get install -y --no-install-recommends \
    git \
    openssh-client \
    && rm -rf /var/lib/apt/lists/*

COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

COPY . .

EXPOSE 8000

CMD ["uvicorn", "app.main:app", "--host", "0.0.0.0", "--port", "8000"]
```

## docker-compose.yml

```yaml
# docker-compose.yml
version: "3.9"

services:
  frontend:
    build:
      context: ./frontend
      dockerfile: Dockerfile
    ports:
      - "80:80"
    depends_on:
      backend:
        condition: service_started
    networks:
      - app-net

  backend:
    build:
      context: ./backend
      dockerfile: Dockerfile
    env_file:
      - .env
    volumes:
      - tech_assistant_data:/data/tech-assistant
      - ${HOST_SSH_DIR:-./ssh}:/root/.ssh:ro   # 可选：挂载全局 git 配置
    depends_on:
      postgres:
        condition: service_healthy
    networks:
      - app-net

  postgres:
    image: postgres:16-alpine
    env_file:
      - .env
    volumes:
      - pg_data:/var/lib/postgresql/data
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U ${POSTGRES_USER} -d ${POSTGRES_DB}"]
      interval: 5s
      timeout: 3s
      retries: 5
    networks:
      - app-net

volumes:
  tech_assistant_data:
    driver: local
    driver_opts:
      type: none
      device: ${HOST_DATA_DIR:-./data/tech-assistant}
      o: bind
  pg_data:
    driver: local
    driver_opts:
      type: none
      device: ${HOST_PG_DIR:-./data/postgres}
      o: bind

networks:
  app-net:
    driver: bridge
```

## 环境变量（.env）

```bash
# === 数据库 ===
POSTGRES_DB=tech_assistant
POSTGRES_USER=tech_user
POSTGRES_PASSWORD=change_this_password
POSTGRES_HOST=postgres
POSTGRES_PORT=5432

# === DeepSeek ===
DEEPSEEK_API_KEY=sk-your_key_here
DEEPSEEK_BASE_URL=https://api.deepseek.com

# === JWT ===
JWT_SECRET_KEY=change_this_secret_key
JWT_ALGORITHM=HS256
JWT_EXPIRE_HOURS=24

# === 主机目录映射（bind mount，数据保存在宿主机） ===
HOST_DATA_DIR=./data/tech-assistant
HOST_PG_DIR=./data/postgres
HOST_SSH_DIR=./ssh
```

## 启动命令

```bash
# 首次启动
docker compose up -d

# 查看日志
docker compose logs -f

# 重建（代码变更后）
docker compose up -d --build

# 停止
docker compose down

# 停止并删除 volume（谨慎，会丢失所有数据）
docker compose down -v
```

## 设计要点

| 决策 | 理由 |
|---|---|
| **bind mount** 替代 named volume | 数据可直接在宿主机查看和备份，开发调试方便 |
| **postgres healthcheck** | 确保 backend 启动时数据库已就绪 |
| **proxy_buffering off** | SSE 流式输出需要禁用 Nginx 缓冲，否则前端收不到实时流 |
| **proxy_read_timeout 300s** | AI 推理可能持续数十秒，需要长超时 |
| **Git 安装在 backend 容器内** | AI 执行 git 命令的依赖 |
| **backend 容器内无 sudo** | 减少安全风险，与沙箱策略一致 |
