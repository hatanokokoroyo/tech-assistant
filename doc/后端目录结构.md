# 后端目录结构

基于 FastAPI + SQLAlchemy + PostgreSQL。

```
backend/
├── app/
│   ├── __init__.py
│   │
│   ├── main.py                      # FastAPI 应用入口（创建 app、注册路由、启动事件）
│   │
│   ├── core/                        # 核心配置
│   │   ├── __init__.py
│   │   ├── config.py                # Pydantic Settings（.env → 配置类）
│   │   ├── security.py              # JWT 生成/验证、密码哈希（bcrypt）
│   │   └── deps.py                  # 公共依赖注入（get_db、get_current_user）
│   │
│   ├── db/                          # 数据库
│   │   ├── __init__.py
│   │   ├── session.py               # SQLAlchemy AsyncSession 配置
│   │   └── base.py                  # ORM Base 声明
│   │
│   ├── models/                      # SQLAlchemy ORM 模型
│   │   ├── __init__.py
│   │   ├── user.py                  # User
│   │   ├── custom_project.py        # CustomProject
│   │   ├── code_repo.py             # CodeRepo
│   │   ├── ssh_key.py               # SshKey
│   │   ├── conversation.py          # Conversation
│   │   ├── message.py               # Message
│   │   └── event_log.py             # EventLog
│   │
│   ├── schemas/                     # Pydantic 请求/响应模型
│   │   ├── __init__.py
│   │   ├── auth.py                  # 注册请求、登录请求、Token 响应、用户响应
│   │   ├── project.py               # 项目创建/更新请求、项目响应
│   │   ├── repo.py                  # 仓库响应、分支响应
│   │   ├── ssh_key.py               # SSH 密钥响应
│   │   ├── file.py                  # 文件目录树、文件内容响应、文件更新请求
│   │   ├── conversation.py          # 对话请求/响应
│   │   ├── message.py               # 消息响应
│   │   └── event.py                 # 事件请求/响应
│   │
│   ├── api/                         # 路由层
│   │   ├── __init__.py
│   │   └── v1/
│   │       ├── __init__.py
│   │       ├── router.py            # 汇总所有路由，注册到 /api/v1
│   │       ├── auth.py              # POST /register, /login; GET/PUT /me
│   │       ├── projects.py          # CRUD /projects
│   │       ├── repos.py             # /projects/{id}/repos/*
│   │       ├── ssh_keys.py          # /ssh-keys/*
│   │       ├── files.py             # /projects/{id}/files/*
│   │       ├── conversations.py     # /projects/{id}/conversations, /conversations/{id}
│   │       ├── stream.py            # POST /conversations/{id}/stream（SSE）
│   │       └── admin.py             # /admin/users
│   │
│   ├── services/                    # 业务逻辑层
│   │   ├── __init__.py
│   │   ├── auth_service.py          # 注册、登录、Token 颁发
│   │   ├── project_service.py       # 项目 CRUD + 目录创建/删除
│   │   ├── repo_service.py          # git clone、分支查询、切换
│   │   ├── file_service.py          # 文件读取/写入（沙箱路径校验）
│   │   ├── conversation_service.py  # 对话 CRUD
│   │   └── event_service.py         # 事件日志生成 + Markdown 文件写入
│   │
│   ├── ai/                          # AI 模块
│   │   ├── __init__.py
│   │   ├── client.py                # OpenAI 客户端封装（DeepSeek 端点配置）
│   │   ├── tools.py                 # Tool Calls 函数实现（read_file, write_file, run_command 等）
│   │   ├── schemas.py               # Tool Calls 的 function calling schema 定义
│   │   └── stream.py                # SSE 流式处理（转发 DeepSeek 流到后端响应）
│   │
│   └── utils/
│       ├── __init__.py
│       ├── path_utils.py            # 路径安全校验（确保在沙箱 /data/tech-assistant 内）
│       ├── git_utils.py             # Git 命令封装（GIT_SSH_COMMAND 注入）
│       └── file_utils.py            # 目录创建、Markdown 文件生成

├── tests/                           # 测试

│

├── requirements.txt                 # 依赖清单
├── .env.example                     # 环境变量模板
└── alembic/                         # 数据库迁移（可选）
    └── ...
```

## 模块分层与职责

```
路由层 (api/v1/*.py)
  │  接收 HTTP 请求，参数校验，调用 service，返回响应
  ▼
服务层 (services/*.py)
  │  业务逻辑编排（事务管理、跨模型操作）
  ▼
模型层 (models/*.py)
  │  SQLAlchemy ORM 定义、数据库查询
  ▼
数据库 (PostgreSQL)
```

AI 模块（`ai/`）被 stream.py 直接调用：

```
POST /api/conversations/{id}/stream
  → stream.py
      → ai/client.py: DeepSeek API 调用
      → ai/tools.py: 执行 Tool Calls 回调
      → ai/schemas.py: function calling 定义
      → SSE 流式响应（逐 token 推送）
```
